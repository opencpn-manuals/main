◆ OVERLAY_LEGACY

#define OVERLAY_LEGACY   0

Overlay rendering priorities determine the layering order of plugin graphics.

These constants define the priority levels for overlay rendering in both OpenGL and standard DC modes. Lower priority values render first (background), while higher priority values render last (foreground).

Plugins should render only when the priority parameter matches their intended layer to ensure proper visual ordering and prevent overlays from appearing above UI elements like toolbars and menus. Default overlay priority for background data visualization.

Used for chart overlays, weather data, tracks, routes, and other navigation information that should appear behind ships and interactive features. Most plugins should render at this priority level.

Examples: GRIB weather overlays, current displays, depth contours
